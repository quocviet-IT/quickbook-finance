# 23. Reports Centre

> Source scope: *QuickBooks Online Business User Manual*, Version 4 - July 2022, pages 117-122.

## Purpose

This chapter explains how QuickBooks Online users open, search, run, customize, save, group, export, email, and automate reports.

It also translates the documented workflow into practical design requirements for an internal accounting and management-reporting platform.

> **Important**
>
> This chapter reflects the QuickBooks Australia workflow documented in July 2022. Current report names, export behavior, scheduling options, permissions, and interface labels may differ in newer versions.

---

## 1. Reports Centre Overview

The Reports Centre provides access to financial, operational, customer, supplier, tax, payroll, and management reports.

Navigate to:

```text
Reports
```

The source manual states that users can:

- open a report immediately;
- search for a report by name;
- add frequently used reports to Favourites;
- print reports;
- email reports;
- export reports to Excel;
- export reports to PDF;
- customize report content and layout;
- save custom reports;
- organize reports into groups;
- schedule report groups for recurring email delivery.

---

## 2. Find and Run a Report

To find a report:

1. Open **Reports** from the left navigation menu.
2. Enter all or part of the report name in the search field.
3. Select the required report.
4. Review the default date range and settings.
5. Run the report.

### Recommended search features

An internal system should support:

- search by report name;
- search by keyword;
- report category filtering;
- recently used reports;
- favourite reports;
- report owner;
- scheduled reports;
- custom reports;
- permission-based visibility.

---

## 3. Frequently Used Financial Reports

The source identifies the following as two of the most frequently produced reports:

- Profit and Loss;
- Balance Sheet.

These reports appear near the top of the All Reports section in the documented interface.

### Profit and Loss

A Profit and Loss report summarizes income and expenses over a period.

```text
Net Profit or Loss
= Total Income
- Cost of Goods Sold
- Operating Expenses
+ Other Income
- Other Expenses
```

Typical report periods include:

- current month;
- previous month;
- quarter;
- year to date;
- financial year;
- custom date range.

### Balance Sheet

A Balance Sheet shows the financial position at a specific date.

```text
Assets = Liabilities + Equity
```

Typical sections include:

- current assets;
- non-current assets;
- current liabilities;
- non-current liabilities;
- equity.

## 4. Favourites

The source manual states that frequently used reports can be added to a **Favourites** area using the star icon.

Recommended favourites behavior:

- favourites are stored per user;
- users can reorder favourites;
- inactive reports are removed or flagged;
- favourites retain report access permissions;
- favourites may link to a saved custom configuration;
- favourite usage is recorded for analytics.

## 6. Customizing Reports

The source states that an open report can be customized using:

```text
Customise
```

Customization areas vary by report and may include:

- General;
- Rows and Columns;
- Lists and Numbers;
- Header and Footer.

After making changes, select:

```text
Run Report
```

This displays the report using the selected configuration.

## 13. Report Groups

The source manual states that users can organize reports that are frequently run together into a Report Group.

### Create a report group

Documented workflow:

1. Select a report.
2. Open its customization options.
3. Select **Save Customisation**.
4. Select **Add new group**.
5. Enter a group name.
6. Select **Add**.
7. Save the report.

### Add more reports to a group

1. Open another report.
2. Select **Save Customisation**.
3. Open **Add this report to a group**.
4. Select the required group.
5. Save.

## 15. Manage Report Groups

The source states that report groups can be viewed from the **Custom Reports** tab.

The Edit menu can be used to:

- export report-group output as PDF;
- export report-group output as Excel;
- delete a report group;
- configure a recurring schedule.

### Recommended group controls

- group name must be unique within the owner scope;
- reports must retain their own parameters;
- deletion should archive rather than erase official groups;
- report order should be configurable;
- access should be permission-based;
- changes should be audited.

## 16. Automated Reports

The source manual states that report groups can be automatically emailed on a recurring schedule.

Example:

```text
First day of every month
```

A schedule may include:

- recipients;
- CC recipients;
- subject;
- email message;
- frequency;
- next delivery date;
- end date;
- attachment format;
- enabled or disabled status.

The source states that recipients do not need to be QuickBooks Online users and QuickBooks does not need to remain open for scheduled reports to be sent.

## 17. Configure a Report Schedule

Documented workflow:

1. Open the **Custom Reports** tab.
2. Select **Edit** beside a report group.
3. Turn on **Set Email Schedule**.
4. Complete the required fields.
5. Select whether to attach the report as Excel instead of PDF.
6. Select **Save and Close**.
7. Return to Custom Reports and review the Email column.

## 22. Report Versioning

Official reports should preserve the configuration used for each generated result.

Store:

- report definition version;
- data snapshot time;
- report parameters;
- accounting method;
- filters;
- currency;
- generated file hash;
- generated by;
- approval status.

Approved or distributed reports should not be silently regenerated with different parameters under the same version.

---

## 23. Reproducibility

A generated report should be reproducible from its stored definition and source snapshot where practical.

Recommended metadata:

```text
Report ID
Definition Version
Generated At
Data Through Date
Filters
Accounting Basis
Currency
Entity Scope
User Scope
File Hash
```

This is especially important for:

- month-end reporting;
- audit evidence;
- statutory reports;
- executive reporting;
- board reporting;
- lender reporting.

---

## 24. Performance and Large Reports

For large data volumes, use:

- asynchronous report generation;
- progress status;
- query timeouts;
- pagination for on-screen results;
- streaming export;
- cached approved snapshots;
- background schedules;
- download expiry;
- retry handling.

The user should not need to keep the report page open while a scheduled or large report is generated.

---

## 25. Risks and Controls

| Risk | Example | Recommended control |
|---|---|---|
| Wrong date range | User sends previous month by mistake | Visible parameters and pre-send review |
| Wrong accounting basis | Cash report used instead of accrual | Store and display basis |
| Sensitive data emailed externally | Payroll report sent to wrong recipient | Recipient approval and DLP validation |
| Stale custom report | Old filters omit new account | Periodic review and versioning |
| Schedule failure | Monthly report is not delivered | Delivery monitoring and retry |
| Duplicate delivery | Retry sends the same report twice | Unique schedule-run key |
| Unauthorized export | User downloads full financial data | Export permission and audit log |
| Formula modification | Excel file changed after export | File hash and approved PDF copy |
| Hidden filters | User misinterprets incomplete report | Active-filter summary |
| Deleted report group | Management pack disappears | Archive and recovery controls |
| Unreproducible result | Current data differs from distributed report | Snapshot and definition version |
| Excessive report load | Large query affects production | Background generation and limits |

## 27. Recommended Constraints

1. Official report definitions must be versioned.
2. Saved report names must be unique within the owner scope where required.
3. Report-group display order must be unique within the group.
4. Schedule recipients must pass email validation.
5. Scheduled runs require a timezone.
6. Generated files must store a hash.
7. Approved report runs cannot be overwritten.
8. Sensitive reports require explicit export and email permissions.
9. Duplicate schedule execution must be prevented by a unique run key.
10. Deleted official report groups should be archived, not permanently removed.
